Filters format

Current version supports filtering only for root objects.

Supported operators

Name Operator
And &
Or |
Equals =
Greater >
Lesser <
Greater or Equals >=
Lesser or Equals <=
Match text match(fieldname,regexp)

Examples

Compare operators

  • booleanField=true
  • integerField<=50
  • doubleField=65.2
  • stringField=Flat
  • dateField=2016-09-05T00:00:00

Compare operators with special symbols.

We need to escape special symbol using tables from HTML URL Encoding Reference
  • tissuePriceFractional=9%2F1 as filter by tissuePriceFractional=9/1
  • damSireName=GRAND TRESOR %28FR%29 as filter by damSireName=GRAND TRESOR (FR)

Match operator

  • match(anyField,*text)
  • match(anyField,text*)

Boolean operators

  • meetingTypeName=Flat|meetingDate=2016-09-05T00:00:00
  • gender=Dog&colour=bk
  • (gender=Dog|colour=bk)&birthDate=2012-10-02T00:00:00
